# PROTOCOLS AND EXTENSIONS

## Objetivo

Aprender la diferencia entre un struct y una classe.

## ¿Qué experimentar?

- Diseñar contratos de comportamiento (similar a las Interfaces en .NET), pero agregarles Extensiones de Protocolo para inyectar una implementación por defecto.
- Extender tipos nativos del sistema (como agregarle una funcionalidad nueva de validación de negocio al tipo String o Double del sistema sin tener que crear clases envolventes o Helpers estáticos).
