# ERROR HANDLING AND RESULT

## Objetivo

Aprender la diferencia entre un struct y una classe.

## ¿Qué experimentar?

- Diseñar un flujo de validación de transacciones que utilice el modelo nativo try / throws.
- Implementar el tipo genérico nativo Result<Success, Failure> para retornar éxitos o errores de manera explícita en APIs o procesos asíncronos, eliminando la posibilidad de que una excepción no controlada derribe el servidor o la aplicación.
